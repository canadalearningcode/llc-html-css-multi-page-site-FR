Élément vide / autofermant

Les balises vides n'affectent pas la mise en page du contenu. Elles intègrent plutôt des objets à la page. Ces balises ne viennent pas en paires, car elles n'habillent pas le contenu comme le font les autres balises que nous avons vues jusqu'à maintenant. Elles sont le contenu!

Comme la syntaxe est légèrement différente dans la version la plus récente du HTML, vous pourriez voir les balises vides écrites de deux manières différentes.

Avant le HTML5, ces balises étaient autofermantes en intégrant une barre oblique vers l'avant directement avant le chevron de droite (/>). En HTML5, les balises vides n'ont pas besoin de s'autofermer.

La balise br (pour break), qui effectue un saut de ligne, est une balise vide : <br /> ou <br>.

<p>Ce texte<br>s'étend sur<br>plusieurs<br>lignes.</p>
